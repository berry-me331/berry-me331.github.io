## CH06 - 객체지향개념

 * 객체 지향 개념의 4가지 특징
   : 캡슐화, 상속, 추상화, 다형성

 * 장점
   : 코드의 재사용성이 높다. 코듸의 관리가 쉽다. 신뢰성 높은 프로그램의 개발이 가능하다


### 클래스와 객체

 * 클래싀
   : 객체를 정의해 놓은것

 - 객체를 생성하는데 사용(설계도 역할)

   * 객체
     : 실제로 존재하는것, 사물 또는 개념

    - 객체의 용도 : 객체의 속성과 기능에 따라 다름(제품 역할)
  
   * 객체와 인스턴스
     : 객체는 인스턴스를 포함하는 의미로, 클래스로부터 생성된 실체를 인스턴스라고 함

   * 객체는 속성과 기능으로 이루어짐(변수, 메서드)
  


### 하나의 소스파일에 여러 클래스 작성

 * 클래스 작성시
   : public이 들어간 main메서드가 들어간건 단 하나의 클래스만 가능. 이게 이름이됨. 이름이 일치해야됨(소스파일 이름과)

    - public이나 main메서드가 없으면 이름을 어느 클래스로 해도상관없음
    - 대소문자를 구분
    - 대소문자가 달라서 실행이 안되면 Run- run confuration?로 실행 클래스 지정가능(오른쪽마우스나 상단통해서)

### 객체 배열

 * 객체 배열
   : == 참조변수 배열

객체 배열을 만들어도, 객체 생성***해줘야됨. 참조변수의 배열임으로, 하나씩 객체 넣어줘야됨

```java
Tv[] tvArr= new Tv[3];

```


### 클래스의 정의

클래스 == 데이텅 + 함수

 * 클래스
   : 설계도, 데이터+함수 , 사용자 정의타입(참조형)

  - 변수 : 하나의 데이터를 저장할 수 있는 공간
  - 배열 : 같은 종류의 여러 데이터를 하나로 저장할 수 있는 공간
  - 구조체 : 서로 관련된 여러 데이터(종류 관계 x)를 하나로 저장할 수 있는 공간
  - 클래스 : 데이터와 함수의 결합(구조체 + 함수)



  * 사용자정의 타입
    : 원하는 타입을 직접 만들 수 있음
    

 ### 선언위치에 따른 변수의 종류

1. 인스턴스 변수 - 클래스 영역 iv : 인스턴스가 생성되었을 때 **
2. 클래스 변수 - 스태틱 클래스 영역 cv : 클래스가 메모리에 올라갈때 생성
3. 지역 변수 - 메서드 영역 lv : 변수 선언문이 수행되었을 때


```java
class Variable{
int iv;//인스턴스 변수
static int cv; //클래스변수
  void method(){
    int lv=;//지역변수
  }
}
```

> 클래스 영역에는 선언문 만가능 ex) 변수선언, 메서드 선언 등 (println 이런건 안됨)
> 변수나 메서드 선언의 순서는 상관없음

 iv 묶어 놓은것들 > 객체
 lv 는 메서드 끝나면 자동 제거

### 메서드

 * 메서드
   : 여러문장을 묶음

 - 메서드는 클래스안에 들어가 있어야됨.
 - 메서드의 장점

    - 중복코드 제거
    - 관리 용이
    - 재사용 가능

- 메서드의 작성 : 반복적으로 수행되는 여러문장을 메서드로, 하나의 메서드는 한가지 기능만 수행하도록 작성

- 메서드
  : 선언부 + 구현부가 나뉨
   - 선언부 : 반환 타입 메서드이름 (타입변수명, 타입변수명 ,,.. 매개변수들)
   - 구현부 : { } 메서드 호출시 수행될 코드들


- 지역변수 : 메서드 내에 선언된 변수 (매개변수 포함)
- 지역변수lv 는 메서드가 끝나면 사라짐


* 메서드의 호출
  : 메서드 이름( 매개변수, 값 등)  < 메서드 호출방법

  < 다른클래스의 메서드 호출 >
  
   1. 클래스 작성
   2. 객체 생성(클래스의)
   3. 객체 사용(메서드 등 사용)


### return 문

 * return 문
   : 실행중인 메서드를 종료하고 호출한 곳으로 되돌아간다

 - 반환타입이 void가 아니면 생략할 수 없음
 - 반환타입이 void가 아니면 return문 !! 반드시 필요
 - if문 안에 썼다면 else나 조건식 거짓일때도 지정해줘야됨 



### 호출 스택( Call stack)

 - 스택 : 밑이 막힌 상자, 위에 차곡차곡 쌓인다
 - 집어넣은 순서대로 아래로 깔리며 쌓임. 꺼낼때도 위에서 부터 꺼낼 수 있음.
 - 아래거를 먼저 꺼낼수 없음. 위에 쌓여야되는걸 중간에 끼워넣을 수도 없음

 - 호출 스택
   : 메서드 수행에 필요한 메모리가 제공되는 공간  
   : 메서드가 호출되면 호출스택에 메모리 할당, 종료되면 해제


- main 메서드가 호출되면, 실행되고,
- 다음 메서드를 호출하면 실행되고 main은 대기, 호출된 메서드가 종료후 main이 다시 실행
- 스택을보면 아래 메서드가 위의 메서드를 호출해서 어떤 메서드가 호출한건지 알수 있음.
- 맨 위의 메서드 하나만 실행중. 나머지는 대기중


### 기본형 매개변수, 참조형 매개변수

<매개변수>

- 기본형 매개변수 - 변수의 값을 읽기만 할 수 있다(read only) > 객체자체의 값을 바꿀수는 없음 (메서드 실행동안에만 바뀜- 지역변수로써)
- 참조형 매개변수 - 변수의 값을 일고 변경할 수 있따.(read & write) > 객체자체에 저장된 값을 바꿔줌

- 참조형 반환타입의 경우 > 객체의 주소/ 객체 자체를 반환함.



### static 메서드와 인스턴스 메서드

* 인스턴스 메서드
   - 인스턴스 생성후, '참조변수.메서드이름()'으로 호출
   - 인스턴스 멤버(iv,im)와 관련된 작업을 하는 메서드
   - 메서드 내에서 인스턴스 변수(iv) 사용 가능
 
* static 메서드(클래스 메서드)
   - 객체생성없이 '클래스이름.메서드이름()'으로 호출 ( 객체 생성을 안해서 참조변수가 없음)
   - 인스턴스 멤버(iv,im)와 관련없는 작업을 하는 메서드
   - 메서드 내에서 인스턴스 변수 사용불가
   - 인스턴스 메서드를 호출할 수 없음.


> iv를 사용하지 않을 때 static을 붙이면 됨


### 오버로딩

 * 오버로딩
   : 한 클래스 안에 같은 이름의 메서드 여러개 정의하는것. 이름은 같지만 매개변수가 다름.

   - 오버로딩 성립 조건
     1. 메서드 이름이 같아야됨
     2. 매개변수의 갯수 또는 타입이 달라야됨
     3. 반환 타입은 영향이 없음




연산자 + 의 오버로딩
: 부호역할, 덧셈역할, 문자열 결합 등




### 생성자

 - 생성자
   : 인스턴스가 생성될때마다 호출되는 '인스턴스 초기화 메서드' (iv묶음)
   : 인스턴스 생성시 수행할 작업(iv 초기화)에 사용


   ```java
   Time t = new Time();
   t.hour = 12;
   t.minute=34;
   t.second=56;

   //위 문장을 한문장으로 쓸 수 있음
   Time t = new Time(12,34,56);   
   ```

 - 생성자 만들때 규칙
   1. 이름이 클래스 이름과 같아야함
   2. 리턴값이 없음(void 안붙임)
   3. 모든 클래스는 반드시 생성자를 가져야함


 - 기본생성자
   : 매개변수가 없는 생성자

   클래스이름 (){ }// 기본생성자

   - 생성자가 하나도 없을때만, 컴파일러가 자동으로 추가해줌
   - 다른생성자가 있으면 꼭 넣어줘야됨(직접)
  
 - 매개변수가 있는 생성자
   : 추가해줘야됨 . 객체 생성과 초기화를 같이할 수 있음
   : new가 객체 생성, 생성자가 초기화



 < 생성자 this() >

 * 생성자 this()
   : 생성자에서 다른 생성자를 호출할때 사용  
   : 다른 생성자 호출시 **반드시 첫줄**에서만 가능함

   ```java
   Car2(){
   this("white","auto",4); //Car2다른 생성자를 호출함

   Car2(String color)
   this(color,"auto",4);//Car2다른 생성자를 호출함

   Car2(String color,String gearType,int door){
   this.color = color;
   this.gearType= gearType;
   this.door=door;
   }
   ```

   - 인스턴스 메서드(생성자포함)에서 사용가능

Car(String c, String g, int d){
